## üìù Note-Taking Best Practices

### Effective Note-Taking
1. **Use Your Own Words:** Don't copy verbatim, rephrase for understanding
2. **Be Visual:** Include diagrams, tables, and code examples
3. **Add Context:** Note why something is important
4. **Link Related Topics:** Cross-reference between notes
5. **Include Examples:** Real code snippets and use cases
6. **Regular Updates:** Revise and improve notes over time

## üéì Learning Frameworks

### Feynman Technique
1. **Choose a concept** you want to understand
2. **Explain it** in simple terms (as if teaching)
3. **Identify gaps** in your explanation
4. **Review and simplify** further

### Cornell Note-Taking Method
- **Cues:** Keywords and questions (left column)
- **Notes:** Main notes during learning (right column)
- **Summary:** Brief summary at bottom
- **Review:** Regular review sessions

### Spaced Repetition Schedule
- **Day 1:** Learn new concept
- **Day 2:** Review
- **Day 7:** Review again
- **Day 30:** Final review
- **As needed:** Refresh when forgotten
