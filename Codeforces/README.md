## üèÖ Contest Strategy

### Before Contest
- [ ] Review common patterns
- [ ] Practice typing speed
- [ ] Set up templates
- [ ] Get familiar with problem types

### During Contest (2 hours)
1. **First 5 minutes:** Read all problems
2. **Problem A (10-15 min):** Quick solve, test carefully
3. **Problem B (20-30 min):** Implement efficiently
4. **Problem C (30-45 min):** Think carefully, then code
5. **Remaining time:** Debug, optimize, or attempt D

### After Contest
- [ ] Read editorial for all problems
- [ ] Upsolve problems not solved during contest
- [ ] Document learnings and patterns
- [ ] Practice similar problem types

## üí° Tips & Best Practices

### Problem Solving
1. **Read Carefully** - Understand constraints and examples
2. **Think Before Coding** - Plan the approach
3. **Start Simple** - Solve examples by hand first
4. **Edge Cases** - n=1, n=max, all same, etc.
5. **Time Complexity** - Calculate before implementing

### Common Pitfalls
- Integer overflow (use `long long` in C++)
- Zero-indexed vs one-indexed
- Not clearing global variables
- Undefined behavior (array bounds)
- Precision errors in floating point

### Contest Mindset
- Don't get stuck on one problem
- Quick submissions for easy problems
- Read all problems first
- Time management is crucial
- Stay calm and focused
