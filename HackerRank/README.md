## ğŸ’¡ HackerRank Strategy

### Problem-Solving Approach

#### Before Solving
1. **Read Carefully** - Understand input/output format
2. **Identify Pattern** - What algorithm/DS is needed?
3. **Plan Approach** - Pseudocode the solution
4. **Consider Edge Cases** - Empty input, large numbers, etc.

#### While Solving
1. **Start Simple** - Get basic solution working first
2. **Test Examples** - Verify with given test cases
3. **Optimize** - Improve time/space complexity
4. **Handle Edge Cases** - Add validation

#### After Solving
1. **Review Editorial** - Learn alternative approaches
2. **Check Discussions** - See others' solutions
3. **Document Learning** - Note patterns and techniques
4. **Practice Similar** - Reinforce the pattern

### Scoring System
- **Easy:** 10-20 points
- **Medium:** 30-50 points
- **Hard:** 60-80 points
- **Expert:** 100+ points

**Stars Requirements:**
- 1 Star: Basic participation
- 2 Stars: 100+ points
- 3 Stars: 500+ points
- 4 Stars: 1500+ points
- 5 Stars: 3000+ points
